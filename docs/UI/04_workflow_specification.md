# Omniverse V2 — Workflow Specification
## Page Compositions

Each workflow is a composition of the components defined in `03_component_specification.md`. This document specifies only what is unique to each page — layouts, column definitions, and workflow-specific interactions. All styling values come from the component spec.

---

## 1. Research Hub — `/research`

**Job:** Select fictional worlds to research, define a global research focus, and initiate research runs.

**Template:** `pages/research.html` — extends `base.html` directly.

### Layout

```
┌──────────────────────────────────────────────────────────────────┐
│ TOOLBAR                                                           │
│ [Focus: ____________________] | [Search______] [⚙️] | [+ Add World] [Start Research] │
├──────────────────────────────────────────────────────────────────┤
│ WORLD TABLE                                                       │
│ [☐] World Name              │ Status              │ Actions      │
│ [☐] The Elder Scrolls       │ Explored ██         │ [Explore] [Research] [✕] │
│ [☐] Warhammer 40k           │ Pending             │ [Explore] [Research] [✕] │
└──────────────────────────────────────────────────────────────────┘
```

### Toolbar Zones

| Zone | Contents |
|---|---|
| Left | Label `Focus:` (`text-[10px] font-bold text-gray-400 uppercase tracking-widest`) + text input (`flex-1 max-w-md`, placeholder "Overall research objective...") |
| Center | Search input (`w-64`, placeholder "Search worlds...") + "⚙️ Filter" button with `aria-haspopup` |
| Right | "Add World" (secondary, opens modal for world import) + "Start Research" (primary) |

Multi-select behavior: "Start Research" button text reflects count when more than one is selected (e.g., "Start Research (3)"), updated by `updateBulkBar()` JS.

**Note:** The "Explore" action in each world row navigates to `/knowledge?world_id={world.uuid}`.

### World Table

Fragment loaded from `/worlds/database-worlds` via HTMX into `#research-world-table`.

Column spec:

| Column | Width | Notes |
|---|---|---|
| Checkbox | 40px | `w-10` — `<th>` has "select all" checkbox (`toggleSelectAll()`). Each `<td>` has `.world-checkbox`. |
| World Name | flex | `font-semibold text-gray-900 dark:text-gray-100 truncate max-w-xs`. UUID shown below in `text-[10px] text-gray-400 font-mono`. |
| Status | 128px | Badge: Explored (`bg-green-100 text-green-700`) or Pending (`bg-gray-100 text-gray-600`). |
| Actions | 112px | Right-aligned. "Explore" (`<a href="/knowledge?world_id=...">`), "Research" (HTMX post), Delete (✕). |

**"Explore"** navigates to the Knowledge Hub filtered to that world.

**"Start Research"** initiates a research run for all checked worlds, using the current Global Focus value. HTMX POST to `/api/worlds/research` with `hx-include="[name='focus']"`.

**"Add World"** opens a modal (`#modal-root`) loading world import/creation form via HTMX.

### Filter — Research

Available fields: World Name, Category, Status, Last Researched. Popup generated by `toggleFilterPopup()` with target `/worlds/database-worlds`.

## 2. Knowledge Hub — `/knowledge`

**Job:** Browse and inspect research artifacts for a selected world.

**Template:** `pages/knowledge.html` — extends `base.html` directly.

**Architecture:** Two-phase flow. No `3_panel.html`. No `active_world` cookie.

### Phase 1: World Selection (no `?world_id`)

```
┌────────────────────────────────────────────────────────────┐
│ TOOLBAR                                                     │
│ [Worlds] [Search worlds...________________]                 │
├────────────────────────────────────────────────────────────┤
│ WORLD LIST (scrollable)                                     │
│ The Elder Scrolls          │ 2026-07-10 │ 42 │ Explored    │
│ Warhammer 40k              │ 2026-07-12 │ 28 │ Explored    │
│ The Culture                │ —          │ 3  │ Pending     │
└────────────────────────────────────────────────────────────┘
```

**Toolbar:**
- Left: "Worlds" label + search input (`w-64`, HTMX search targeting `#knowledge-content` via `/knowledge/worlds/list`)
- Right: loading indicator only

**World List** (`components/knowledge_world_list.html`):
- Rows: world name, franchise subtitle, last researched date, artifact count badge, explored/pending badge
- Clicking a row calls `loadWorldDetail(worldId)` → navigates to `/knowledge?world_id={uuid}`

### Phase 2: World Detail (`?world_id={uuid}`)

```
┌──────────────────────────────────────────────────────────────────────┐
│ TOOLBAR                                                               │
│ [← Back to worlds] | [World Name]                                     │
├──────────────────────────────────────────────────────────────────────┤
│ TAB BAR                                                               │
│ [Overview] [Artifacts] [Notebook] [Theory]   [Start Focused Research] │
├──────────────────────────────────┬───────────────────────────────────┤
│ TAB CONTENT                       │ INSPECTOR (320px, hidden by       │
│ (flex: 1, scrollable)            │  default, shown on item click)    │
│                                   │                                   │
│ Loaded via HTMX from              │ Artifact detail or notebook       │
│ /knowledge/world/{id}/tab/{tab}   │ entry detail                      │
└──────────────────────────────────┴───────────────────────────────────┘
```

**Toolbar:**
- Left: "← Back to worlds" button (`onclick="window.location.href='/knowledge'"`) + world name
- Right: nothing (Start Focused Research is in the tab bar)

**Sub-tabs** (rendered in `components/knowledge_world_detail.html`):

| Tab | Content Template | Description |
|---|---|---|
| Overview | `knowledge_overview_tab.html` | World header (name, summary, explored badge), 4-card stats grid (Artifacts, Claims, Notebook, Theories), recent artifacts list, raw data, theory entry |
| Artifacts | `artifact_list.html` | Artifact list with type badges, click to load inspector |
| Notebook | `knowledge_notebook_tab.html` | Notebook entries (click for detail in inspector) + Notebook claims |
| Theory | `knowledge_theory_tab.html` | Theory text + auditor feedback, or "Generate Theory" prompt |

**Sub-tab switching:** `switchWorldTab()` JS updates active tab styling and loads content via `GET /knowledge/world/{id}/tab/{tab_name}` into `#world-tab-content`.

**Inspector:** Right-side panel (`w-80`) hidden by default. Revealed by `showInspector(url)` or `loadNotebookDetail(entryId)`. Content loaded via HTMX.

**"Start Focused Research":** Primary button in tab bar, POST to `/api/worlds/research`.

### Filter — Knowledge

Currently, the Knowledge page does not implement the filter popup (unlike Research and Execution). Search is via the toolbar search input only.

## 3. Execution Hub — `/logs`

**Job:** Monitor active and recent research runs. View logs. Abort individual runs or all runs.

**Template:** `pages/logs.html` — extends `base.html` directly.

### Layout

```
┌───────────────────────────────────────────────────────────────────────┐
│ TOOLBAR                                                                │
│ [Filter runs...__________] [⚙️]                             │ View All Logs │ Abort All │
├───────────────────────────────────────────────────────────────────────┤
│ RUN TABLE (auto-refresh every 5s)                                     │
│ World       │ Focus           │ Progress        │ %    │ Actions      │
│ Elder Scrolls │ Divine Artifacts │ ██████░░░░ │ 65%  │ Logs | Abort │
└───────────────────────────────────────────────────────────────────────┘
```

### Toolbar Zones

| Zone | Contents |
|---|---|
| Left | Search input (`w-64`, placeholder "Filter runs...", HTMX search targeting `#execution-table`) + "⚙️ Filter" button |
| Center | Empty (`flex-1` spacer) |
| Right | "View All Logs" (secondary) + "Abort All" (danger filled, `hx-confirm` confirmation dialog) |

### Run Table

Fragment loaded from `/api/runs/active-detailed` via HTMX, auto-refreshed every 5 seconds. `updateTimestamp()` updates a "last updated" label after each swap.

Column spec:

| Column | Width | Notes |
|---|---|---|
| World | flex | `font-weight: 600` |
| Focus | flex | `color: var(--text-secondary)` |
| Progress | 150px | Inline progress bar component |
| % | 50px | Centered, `font-weight: 600` |
| Actions | auto | "Logs" + "Abort" buttons |

**Progress color rules:**
- 0–100% active run: `#2563eb` (blue)
- Stalled or very slow: `#f59e0b` (amber)
- Completed: `#16a34a` (green)

**"Logs"** loads the full log view into the table area.

**"Abort"** sends abort signal to that specific run.

**"Abort All"** aborts all active runs — `hx-post="/api/runs/abort-all"` with `hx-confirm` for confirmation.

**"View All Logs"** loads the complete log list via `GET /logs/list`.

### Filter — Execution

Available fields: Status (Running / Stalled / Completed / Aborted), World, Progress (> / <). Popup generated by `toggleFilterPopup()` with target `/api/runs/active-detailed`.

## 4. Settings Hub — `/settings`

**Job:** Configure AI providers, routing, application globals, and view system health.

**Template:** Settings extends `base.html` via `layout/3_panel.html`.

### Layout

```
┌───────────────────────────────────────────────────────┐
│ TAB BAR                                               │
│ [General] [Providers*] [Routes] [Health]              │
├─────────────────────────┬─────────────────────────────┤
│  (tab-specific content) │  (tab-specific content)     │
└─────────────────────────┴─────────────────────────────┘
```

The Settings page replaces the standard toolbar with a tab bar. All content is loaded via HTMX fragment swaps targeting the tab panel region.

---

### 4.1 General Tab

**Layout:** Single scrollable form panel with section headers.

**Sections:**

**General Configuration**
2-column grid layout:

| Field | Type | Notes |
|---|---|---|
| Application Name | text input | |
| Database URL | text input | |
| Default Research Model | select | Options from available provider models |

**Quick Provider Access** (read-only summary table)

| Column | Notes |
|---|---|
| Provider | Provider name |
| Model | Primary model |
| Status | "● Active" in `--status-ok` or "● Error" in `--status-error` |

**Footer action bar** (fixed to bottom of panel):
- Height: 52px, `background: var(--bg-surface)`, `border-top: 1px solid var(--border)`
- Right-aligned: "Cancel" (secondary) + "Save Changes" (primary)

---

### 4.2 Providers Tab

**Layout:** List (300px) + Inspector (flex: 1) split.

**Provider List:**

Header row: "PROVIDERS" label (field label style) + "+ New" button.

Each list item:
- Name: `font-weight: 600`, `font-size: 0.8rem`
- Subtitle: `provider_type / first two models...`, `font-size: 0.7rem`, `color: var(--text-secondary)`
- Active item: `background: var(--bg-active)`, `border-left: 3px solid var(--accent)`

**Provider Inspector:**

Title row: Provider name (`font-weight: 800`, `1.1rem`) + "Delete" (secondary) + "Save Provider" (primary), right-aligned.

Form fields:

| Field | Layout | Notes |
|---|---|---|
| Provider Type | 2-column grid (left) | e.g., `openai`, `anthropic`, `ollama` |
| Base URL | 2-column grid (right) | e.g., `https://api.openai.com/v1` |
| Supported Models | Full width, with inline button | CSV string + "🔄 Sync" button |

**🔄 Sync button:**
- Appears inline to the right of the Supported Models input
- Secondary style, `height: 30px`, `padding: 0 12px`
- `title="Fetch models from provider API"`
- On click: button shows loading state ("Syncing..."), sends `POST /settings/providers/{id}/sync-models`, updates the input with returned model list

**API Keys table:**

Preceded by: "API KEYS & PRIORITY" label (field label style) + "+ Add Key" button (right-aligned).

| Column | Width | Notes |
|---|---|---|
| Key | flex | Inline text input, value masked as `sk-proj-...` |
| Priority | 80px | Inline numeric input, centered |
| Action | 60px | "Delete" ghost button in `--danger` |

---

### 4.3 Routes Tab

**Layout:** List (300px) + Inspector (flex: 1) split.

**Route List:**

Each item represents a task type (agent identifier).
- Name: task type label, `font-weight: 600`
- Subtitle: summary of provider chain (e.g., "OpenAI-Main → Anthropic-Pro")

**Route Inspector:**

Title: Task type name.

**Provider Chain** (the core UI element of this tab):

Each row in the chain:

| Column | Notes |
|---|---|
| Priority | Number input (order in chain) |
| Provider | Select from configured providers |
| Model | Select from that provider's supported models |
| Action | "Remove" ghost button |

"+ Add Provider to Chain" — secondary button below the table.

**Override Default Routes** — checkbox at bottom. When checked, this route config replaces the global default.

---

### 4.4 Health Tab

**Layout:** Single scrollable panel.

**Model Status Matrix:**

A table where rows = providers and columns = models. Each cell shows the live status of that provider/model combination.

| Provider | gpt-4o | gpt-4-turbo | claude-3-5 |
|---|---|---|---|
| OpenAI-Main | ✅ Active | ✅ Active | — |
| Anthropic-Pro | — | — | ✅ Active |
| Local-Ollama | — | — | — |

Status cell values: "✅ Active" (`--status-ok`), "⚠️ Degraded" (`--status-warn`), "❌ Down" (`--status-error`), "—" (not configured).

**Snapshot Manager:**
- Shows: "Last checked: [timestamp]"
- "Refresh" button (secondary) — triggers a new health check against all configured providers
